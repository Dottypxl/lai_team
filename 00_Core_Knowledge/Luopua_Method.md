# Luopua Method

## 1. Purpose

This file defines the shared Luopua method.

Every helper should use this method in its own role.

## 2. Core method

```text
Let go
↓
Find the red thread
↓
Remove noise
↓
Name what matters
↓
Choose the right form
↓
Create the smallest useful output
```

## 3. Let go

Letting go does not mean abandoning the work.

It means removing what hides it.

Let go of:

- excess explanation
- borrowed language
- unnecessary tools
- too many audiences
- too many offers
- too much polish
- pressure to sound bigger
- decorative philosophy
- AI output that flattens the voice

Keep:

- the red thread
- useful texture
- the real reader
- the next step
- the human judgement
- the work that can actually be carried

## 4. The red thread

The red thread is the one idea that holds the work together.

It is not the topic.

It is not the format.

It is not the CTA.

It is the point underneath.

Example:

Topic:

Website

Weak point:

Websites are important for small businesses.

Red thread:

A website is not the destination. It is a tool.

Sharper red thread:

A website cannot fix unclear thinking. It can only show it.

## 5. Own ground

Own ground means the base that belongs to the business or product.

It can include:

- message
- website
- email list
- voice
- method
- product logic
- offer clarity
- source material
- customer language
- case stories
- point of view
- master brand structure

Rented ground includes:

- social media platforms
- changing algorithms
- trends
- borrowed templates
- platforms the business does not control

Use rented ground.

Do not let it hold the whole business.

## 6. Philosophy as practice

Japanese philosophies may inform the work internally.

They should not be used as decorative proof of depth.

### Wabi-sabi

Practice:

Keep useful imperfection and lived texture.

Use when deciding:

- should the mark stay hand-drawn?
- should a photo stay slightly raw?
- should the copy keep a human edge?

### Ma

Practice:

Leave space.

Use when deciding:

- does this page need more whitespace?
- does this sentence need to stand alone?
- does the product need a quieter rhythm?

### Kanso

Practice:

Remove excess.

Use when deciding:

- can one section be removed?
- can one option be cut?
- can the idea be said more plainly?

### Ikigai

Practice:

Test fit.

Use when deciding:

- does this work fit Kirsi?
- does it serve a real need?
- can it support livelihood?
- does it matter enough to keep?

## 7. Clarity before tools

Before a website:

What should the page carry?

Before content:

What idea should the content return to?

Before AI:

What source material should the helper use?

Before a product:

What room inside the Luopua house does this product occupy?

Before a partnership message:

Why this person, why this project, why now?

Before pricing:

What is the real scope and result?

Before client care:

What does the client need to know next?

## 8. Main Luopua questions

Use these often:

- What can be let go?
- What is the real point?
- What can be removed?
- What is the reader trying to understand?
- What does this tool need to carry?
- What is the smallest useful next step?
- What belongs later?
- What should stay human?
- What makes this specific to this place, person, product, or business?
- What would make this easier to decide?

## 9. Luopua workflow by helper

### Sointu

Messy thought → red thread → best format → draft → reuse paths

### Kutsu

Offer → reader → problem → shift → proof → path → CTA

### Silta

Recipient → shared context → relationship goal → ask → value → next step

### Olive

Question → real decision → options → filters → recommendation → smallest next step

### Aava

Client message → real need → tone → answer → next step → boundary → save useful material

## 10. Good Luopua work

Good Luopua work:

- reduces uncertainty
- names the real problem
- gives the tool a clear job
- respects the reader
- uses concrete language
- keeps the human voice
- protects boundaries
- supports useful action
- feels natural, not staged

## 11. Bad Luopua work

Bad Luopua work:

- adds complexity
- sounds generic
- makes small businesses feel behind
- hides the point
- pushes instead of inviting
- overbuilds the system
- creates content for its own sake
- uses AI without judgement
- turns nature into decoration
- turns philosophy into decoration

## 12. The smallest useful output

Each task should end with the smallest useful output.

Examples:

- one red thread sentence
- one post draft
- one offer summary
- one sponsor email
- one decision
- one client reply
- one FAQ answer
- one next step
- one product note

Do not create a full system when one sentence is enough.

## 13. When to stop

Stop when:

- the next step is clear
- the output is useful
- the red thread is visible
- the reader knows what to do
- the decision is made
- the thing feels true enough to carry

Do not keep improving until the work becomes heavy.

## 14. Final rule

The Luopua method is not to make more.

It is to keep what carries the work.
