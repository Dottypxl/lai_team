## 1. Purpose

This file gives Sointu reusable carousel patterns for Luopua content.

A carousel should not be a pile of tips.

It should move the reader from one way of seeing to another.

The best Luopua carousel creates a shift:

```text
What the reader thinks
↓
What is really happening
↓
What needs to become clearer
↓
What to do next
```

## 2. Carousel rules

Use:

- one idea per slide
- short slide text
- clear movement
- strong contrast
- grounded language
- a final slide that gives the reader something to keep

Avoid:

- too much text
- generic tips
- fake urgency
- vague inspiration
- “5 ways to grow your brand”
- slides that repeat the same idea
- captions that explain what the slides already said

## 3. Default Luopua carousel structure

```text
Slide 1: Strong opening
Slide 2: Familiar belief
Slide 3: Hidden problem
Slide 4: Shift
Slide 5: Practical example
Slide 6: What to ask next
Slide 7: Takeaway
```

## 4. Pattern: Surface problem vs real problem

Use when people think they need one thing, but the real work is elsewhere.

### Template

Slide 1:

You may not need [surface solution].

Slide 2:

It is easy to think [surface solution] will fix [problem].

Slide 3:

But often the real issue is [deeper issue].

Slide 4:

[Surface solution] can only carry what is already clear.

Slide 5:

If [example of unclear thing], then [surface solution] will repeat it.

Slide 6:

Before [surface solution], ask:

[clarifying question]

Slide 7:

Build the clarity first.

Then build the tool.

Slide 5:

Slide 6:

Before the website, ask:

Slide 7:

Build the clarity first.

Then build the tool.

## 13. Carousel quality check

Before sending a carousel, check:

- Does slide 1 make a clear promise?
- Does each slide carry only one thought?
- Is there a real shift?
- Is the language concrete?
- Could this belong only to Luopua?
- Is there too much text?
- Does the final slide land?
- Can one slide be removed?
- Does the caption add context instead of repeating the slides?

## 14. Final rule

A Luopua carousel should not teach for the sake of teaching.

It should help the reader see the real problem with less fog.
