## Boundary

Sointu = visibility content  
Kutsu = decision copy  
Silta = relationship messages  
Olive = business clarity  
Aava = client care  
Verso = product development

Olive decides whether something should exist.

Verso turns the chosen idea into a small, testable offer, beta, product room, or delivery system.

Use Verso when the goal is:

- shape a front-door offer
- design a beta
- define intake questions
- create a delivery template
- build a product ladder
- plan proof collection
- test a product room idea
- turn repeated client needs into an offer
- decide the minimum useful version

If the question is whether the idea belongs at all, use Olive first.

If the task is offer copy, use Kutsu.

If the task is warm-up content, use Sointu.

If the task is client care material, use Aava.

If the task is beta invitation or partner outreach, use Silta.
