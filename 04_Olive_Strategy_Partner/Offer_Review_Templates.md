# Olive – Offer Review Templates

## 1. Purpose

This file gives Olive templates for reviewing offers.

Use these before Kutsu writes copy.

If the offer is unclear, copy will only make the confusion public.

## 2. Offer review template

```text
Offer name:
[Name]

Buyer:
[Who this is for]

Problem:
[What problem this reduces]

Result:
[What becomes clearer or easier]

Deliverables:
[What the client receives]

Process:
[Steps]

Time:
[How long this takes]

Price:
[Price or price range]

Boundary:
[What is not included]

Proof:
[Why Kirsi can do this]

Next step:
[How the client starts]
```

## 3. Offer scorecard

Score each item from 1 to 3.

```text
1 = unclear
2 = partly clear
3 = clear enough
```

Items:

- buyer
- problem
- result
- deliverables
- process
- price
- boundary
- proof
- next step

Decision:

```text
24–27 = ready for Kutsu copy
18–23 = clarify before copy
under 18 = not ready as an offer
```

## 4. Luopua Clarity Session review

### Offer name

Luopua Clarity Session

### Buyer

Small or rural business owner whose message, website, content, or AI source material feels scattered.

### Problem

The business has real work behind it, but the words do not yet carry it clearly.

### Result

The client leaves with one practical clarity base.

### Deliverables

- red thread sentence
- audience notes
- offer summary
- key message angles
- next-step recommendation
- short source text for website, content, or AI work

### Process

1. Client sends messy material.
2. Kirsi reviews it.
3. Session is held.
4. Kirsi creates a one-page clarity base.
5. Client uses it for the next build.

### Boundary

Not a full brand strategy.

Not a website build.

Not monthly content.

Not unlimited AI support.

### Price

Set only after time and scope are confirmed.

### Next step

Book a clarity session or send messy material for fit check.

## 5. Website Clarity Review template

### Offer name

Website Clarity Review

### Buyer

Small business owner whose website exists, but does not explain the work clearly enough.

### Problem

The website may look acceptable, but the reader path, offer, or message is unclear.

### Result

The client knows what to fix first.

### Deliverables

- homepage clarity notes
- offer clarity notes
- reader path notes
- priority fix list
- next-step recommendation

### Boundary

Not a full redesign.

Not a technical audit unless agreed.

Not full copy rewrite.

### Next step

Send website link.

## 6. AI Helper Setup template

### Offer name

AI Helper Setup

### Buyer

Small business owner or solo operator who wants AI support without generic output.

### Problem

AI tools produce vague, wrong, or off-brand material because they lack role clarity and source material.

### Result

The client has one AI helper with a clear role, voice boundaries, and workflow.

### Deliverables

- helper role instructions
- voice boundaries
- one workflow file
- source file list
- example prompt
- usage notes

### Boundary

One helper only, unless larger scope is agreed.

No full automation build.

No open-ended support.

### Next step

Choose one business task for the helper.

## 11. Offer readiness questions

Ask:

- Can I explain this offer in one sentence?
- Can the buyer recognize themselves?
- Does the offer solve one clear problem?
- Does the result feel concrete?
- Are deliverables named?
- Is the boundary clear?
- Is the price connected to real work?
- Is the next step easy?

## 12. Before sending to Kutsu

The offer is ready for Kutsu when Olive can provide:

```text
Offer name:
Buyer:
Problem:
Result:
Deliverables:
Process:
Boundary:
Price:
CTA:
```

If these are missing, do not write the page yet.

## 13. Final rule

A clear offer makes copy easier.

If copy feels hard, the offer may still be unclear.
